# Phase 4 Plan 01: AI-Assisted Selling Workflow

## Objective
- **What:** Improve AI generation reliability and complete the end-to-end admin selling flow (generate -> edit -> save -> copy caption/public listing link -> store/remove Facebook post link).
- **Why:** AI/social workflow is the core productivity differentiator of the product.
- **Output:** Reliable AI-driven listing content pipeline with coherent Facebook posting records and up-to-date API/docs.

## Verification
1. AI description and FB caption generation both pass happy-path and failure-path tests.
2. Admin can complete AI import to saved item with manual edits preserved.
3. AI draft import failure modes do not leave orphaned storage/database state and surface actionable errors.
4. FB post history in item detail, items list, and FacebookPosts page remains consistent after save/add/delete actions.
5. Admin can copy generated caption and the public item listing link required by `AISO-02` without losing saved content.
6. API errors from AI providers surface as normalized domain errors.
7. `docs/API_CONTRACT.md` matches the final Phase 4 request/response behavior.
