# Phase 15 Plan 01: Schema — Platform Role & Shop Role Extension

## Objective

- **What:** Thêm biểu diễn **quyền platform** trên `User`, mở rộng `ShopRole` cho tenant (tối thiểu `shop_staff`), migration + backfill an toàn.
- **Why:** Tách semantic “super toàn hệ thống” khỏi bản ghi shop; chuẩn bị guard và API ở plan 02.
- **Output:** Prisma schema + migration SQL đã review; tài liệu contract cập nhật phần auth/payload nếu có field mới cho client.

## Design detail (executor must implement consistently)

1. **`PlatformRole` enum** (tên chính xác có thể `platform_super` / `none` — chốt trong schema):  
   - `User.platform_role PlatformRole?` default `null` hoặc `none`.  
   - Semantics: chỉ user có giá trị platform mới được coi là **platform operator** sau khi legacy path tắt.

2. **`ShopRole` enum** — additive:  
   - Giữ `super_admin`, `shop_admin` tạm thời **hoặc** rename có bước migration map (ưu tiên **additive** `shop_staff` để giảm risk).  
   - Nếu giữ `super_admin` trong enum tenant: document rằng route platform sẽ không còn rely vào row này sau plan 02 (chỉ migration backfill `platform_role`).

3. **Backfill SQL** (trong migration hoặc script follow-up):  
   - `UPDATE users SET platform_role = 'platform_super' WHERE id IN (SELECT DISTINCT user_id FROM user_shop_roles WHERE role = 'super_admin');`  
   - Không xóa row `user_shop_roles` trong migration đầu (tránh surprise); plan 02 có thể optional cleanup / demote.

4. **`ShopAuditAction`** — thêm giá trị enum cho: `set_platform_role`, `set_shop_member_role` (tên snake_case khớp convention hiện có).
