# Phase 14 Plan 02: API scoping, tenant guard và shop management endpoints

## Objective
- **What:** Implement tenant context trong JWT, NestJS guard enforce isolation, thêm switch-shop endpoint và CRUD shop cho super admin.
- **Why:** MULT-02 và MULT-03 — admin chỉ thấy data của shop họ đang active; API phải cứng không thể bypass.
- **Output:** `TenantGuard`, decorator `@CurrentTenantId()`, endpoint `POST /auth/switch-shop`, CRUD `/admin/shops`.

## Context
@.planning/REQUIREMENTS.md

## Tasks

<task type="code">
  <name>Thêm switch-shop vào AuthService và AuthController</name>
  <files>backend/src/auth/auth.service.ts, backend/src/auth/auth.controller.ts, backend/src/auth/dto/switch-shop.dto.ts</files>
  <action>
    Thêm `POST /auth/switch-shop` nhận `{ shop_id: string }`.
    AuthService kiểm tra user có record trong `user_shop_roles` với `shop_id` đó.
    Nếu hợp lệ: issue JWT mới có thêm field `active_shop_id`.
    Nếu không hợp lệ: throw 403 FORBIDDEN với error code `AUTH_FORBIDDEN`.
    Cập nhật `JwtPayload` interface/type để có `active_shop_id?: string`.
  </action>
  <verify>
    Unit test `auth.service.spec.ts`:
    - Case hợp lệ: token mới có `active_shop_id`.
    - Case không có role: trả 403.
    - Case shop không tồn tại: trả 404.
  </verify>
  <done>Switch-shop endpoint hoạt động và test coverage đạt.
  </done>
</task>

<task type="code">
  <name>Tạo TenantGuard và decorator @CurrentTenantId()</name>
  <files>backend/src/common/guards/tenant.guard.ts, backend/src/common/decorators/tenant.decorator.ts</files>
  <action>
    Tạo `TenantGuard` implements `CanActivate`.
    Guard đọc `active_shop_id` từ JWT payload đã được decode bởi AuthGuard.
    Nếu `active_shop_id` không có: throw 400 với message "No active shop context".
    Gán `request.tenantId = active_shop_id`.
    Tạo decorator `@CurrentTenantId()` đọc từ `request.tenantId`.
  </action>
  <verify>
    Unit test guard:
    - JWT có `active_shop_id`: pass, gán tenantId.
    - JWT không có `active_shop_id`: throw 400.
    - Không có JWT: throw 401 (handled bởi AuthGuard trước).
  </verify>
  <done>Guard bảo vệ đúng; decorator đọc tenantId chuẩn.</done>
</task>

<task type="code">
  <name>Scope ItemsService theo tenantId</name>
  <files>backend/src/items/items.service.ts, backend/src/items/items.controller.ts</files>
  <action>
    Mọi method query trong `ItemsService` (findAll, findOne, create, update, remove) đều thêm `shop_id: tenantId` vào where clause.
    Trong controller, thêm `@UseGuards(TenantGuard)` và `@CurrentTenantId() tenantId` để truyền vào service.
    Kiểm tra `findOne`: nếu item.shop_id !== tenantId → throw 404 (không lộ thông tin cross-tenant).
  </action>
  <verify>
    Unit test `items.service.spec.ts`:
    - Tạo 2 item với 2 shop khác nhau.
    - findAll với tenantId của shop 1 → chỉ trả item của shop 1.
    - findOne với item của shop 2 bằng tenantId shop 1 → 404.
  </verify>
  <done>Items service hoàn toàn isolated theo tenant.</done>
</task>
