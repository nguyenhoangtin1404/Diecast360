# Phase 14 Plan 01: Multi-tenant schema và data isolation

## Objective
- **What:** Mở rộng Prisma schema để hỗ trợ multi-tenant. Tạo bảng `shops`, `user_shop_roles`, thêm `shop_id` vào `items`.
- **Why:** MULT-01: Mỗi shop là một tenant riêng biệt — data không được chia sẻ giữa các tenant.
- **Output:** Migration an toàn, schema validate thành công, `prisma generate` không lỗi.

## Context
@.planning/REQUIREMENTS.md
@.planning/phases/14-chuc-nang-quan-ly-shop-multi-tenant/14-CONTEXT.md

## Tasks

<task type="code">
  <name>Thêm model Shop và UserShopRole vào schema.prisma</name>
  <files>backend/prisma/schema.prisma</files>
  <action>
    Thêm model `Shop` (id, name, slug unique, is_active, timestamps).
    Thêm model `UserShopRole` với composite PK (user_id, shop_id), role enum `ShopRole { super_admin shop_admin }`.
    Thêm relation từ `User` → `UserShopRole[]` và `Shop` → `UserShopRole[]`.
  </action>
  <verify>npx prisma format && npx prisma validate</verify>
  <done>Schema hợp lệ, không có lỗi, relation hai chiều đúng.</done>
</task>

<task type="code">
  <name>Thêm cột shop_id vào bảng items với migration strategy an toàn</name>
  <files>backend/prisma/schema.prisma, backend/prisma/migrations/</files>
  <action>
    Thêm `shop_id String` vào model `Item` với FK → `Shop`.
    Tạo migration theo 2 bước:
    1. Thêm `shop_id` nullable trước.
    2. Tạo seed shop mặc định, UPDATE tất cả items cũ có `shop_id IS NULL`.
    3. ALTER COLUMN thành NOT NULL.
    Lưu ý: theo nguyên tắc trong DB_SCHEMA.md — không sửa migration cũ, tạo file mới.
  </action>
  <verify>
    - npx prisma migrate dev --name add_multi_tenant_shop
    - npx prisma db seed (nếu có seed file)
    - Kiểm tra không record nào có shop_id IS NULL sau migration
  </verify>
  <done>Migration apply thành công, mọi item cũ có shop_id hợp lệ.</done>
</task>
