- **Why:** Transitions the project from feature-complete to production-ready with automated social publishing.
- **Output:** Reproducible local runtime, enforceable CI quality gates, and automated Facebook posting endpoint.

## Completed Tasks

### 1. Docker / Operational Baseline ✅

### 2. CI Hardening ✅

- `ci-success` job gates on `backend` + `frontend` (security audit is advisory).

### 3. Facebook Graph API Integration ✅

**New module:** `backend/src/integrations/facebook/`

| File | Role |
|------|------|
| `facebook-config.service.ts` | Validates `FACEBOOK_PAGE_ID` + `FACEBOOK_PAGE_ACCESS_TOKEN` at request time |
| `facebook-graph.service.ts` | Calls `POST /{page-id}/feed`, maps Graph API errors to `AppException` |
| `facebook.module.ts` | NestJS module exporting both services |
| `index.ts` | Barrel export |

**Error code mapping:**

| Graph API code | App error code | HTTP |
|----------------|---------------|------|
| 190 | `FACEBOOK_AUTH_ERROR` | 401 |
| 200 | `FACEBOOK_PERMISSION_ERROR` | 403 |
| 4, 32 | `RATE_LIMIT_EXCEEDED` | 429 |
| Other | `FACEBOOK_PUBLISH_ERROR` | 502 |

**Endpoint:** `POST /api/v1/items/:id/facebook-posts/publish`
- DTO: `PublishFacebookPostDto` — optional `content` override.
- Resolves caption: `dto.content` → `item.fb_post_content` → throws if empty.
- Persists result as `facebook_post` record (same table as manual flow).
- Rate limited: 5 req/min.
